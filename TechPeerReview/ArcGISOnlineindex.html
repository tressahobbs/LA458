Emma’s technology report on ArcGIS Online was delivered in a very understandable manner. I appreciate that each slide in the PowerPoint presentation was clearly labeled with a consistent font, so it was extremely easy to identify the sub-topic for each slide. I also liked that the summary slide had a different background from the rest of the presentation. It grabbed my attention and let me know it was the final slide with information before the references slide without having to put the word “summary” in the title. Content-wise, I think the report was very informative. It walked the viewer through creating a simple map step by step and included both written descriptions and visual images of each step. I also like that APIs and SDKs were addressed and defined early in the report because it allowed me to understand some of the backend work to using ArcGIS Online. I thought that the example project was good for a basic map and the design is clear and concise. I also like that Emma chose to use hospital data because it made the project relevant to what is happening in the world currently, which in my opinion always makes a project more interesting.


While the technology report was very informative, there were a few things that I would like to know that weren’t covered in the PowerPoint presentation. Mainly, I would like to know how some smaller processes within the overall map creation work. It would have been nice to see how some things like configuring pop-ups and changing the symbology of the map could tailor the map for a particular purpose and provide more detailed information or clean up the data depending on the default presentation of the data. I would have also loved to have seen the function ArcGIS Online has to turn a map into a web mapping application since the class focuses on web mapping. It would have been interesting to see two different ways of creating a web map, one being using the ArcGIS Online web application function and the other creating web maps using a text editor, such as GitHub or, leaflet.js, and then comparing and contrasting the two processes. It also would’ve been nice to see how this technology could be used for other case studies, whether through another map example or just a narrative describing how the technology could be applied to a different case study.


Based on what I now know about ArcGIS Online, I think that I can apply this technology to a lot of projects that I am currently doing or plan to do in the future. I want to take the data that I have been collecting in another one of my classes on the satisfaction of public spaces in Ames, as well as preferred transportation type and commute time, and input it into ArcGIS Online. I’d do this by creating a csv. file containing my data and then upload it into ArcGIS Online where I could run a suitability analysis to figure out which public places need to be improved and map the results. Then I could turn the resulting map output into a web map to share with the community.
